startTileGrid

@JvmOverloads()
final static TileGrid startTileGrid(AppConfig config, EventBus eventBus)
Content copied to clipboard

Builds a new Application using the given parameters, calls Application.start on it and returns its TileGrid only. Use this function if you don't care about how the Application works and you don't want to manage its lifecycle.

This factory method uses sensible defaults, and it is fine to call it without parameters.

Note thatstartTileGrid will overwrite the Application that the TileGrid is currently using.

Also make sure that all the objects you pass use the same AppConfig.
